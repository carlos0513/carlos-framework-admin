## 1. 思维转换：后端 → 前端

### 1.1 核心差异对照表

| 维度       | 后端思维                 | 前端思维                     | 类比理解                          |
|----------|----------------------|--------------------------|-------------------------------|
| **状态管理** | 数据库持久化               | 内存状态 + 本地存储              | 像 Redis，但会随页面关闭丢失             |
| **生命周期** | 请求-响应周期              | 组件挂载-更新-卸载               | 类似 Servlet 生命周期               |
| **路由**   | URL 映射到 Controller   | 路由映射到组件                  | 类似 Spring MVC @RequestMapping |
| **依赖注入** | Spring IoC 容器        | Provide/Inject 或全局 Store | 类似 Spring 的 @Autowired        |
| **模板渲染** | Thymeleaf/Freemarker | Vue Template/JSX         | 类似 JSP，但更强大                   |
| **数据流**  | 单向：请求 → 处理 → 响应      | 双向绑定 + 单向数据流             | 需要适应响应式编程                     |

### 1.2 响应式编程核心概念

后端工程师最容易困惑的是**响应式数据**。把它想象成：

```
┌─────────────────────────────────────────────────────────────┐
│  后端视角：数据变了，需要手动通知前端刷新                        │
│  前端视角：数据变了，视图自动更新（像 Excel 公式）               │
└─────────────────────────────────────────────────────────────┘
```

**关键代码对比**:

```java
// 后端：Java - 数据变化不会自动触发任何操作
private String userName = "admin";
public void updateName(String name) {
    this.userName = name;
    // 需要手动调用其他方法
}
```

```typescript
// 前端：Vue3 - ref 让数据变成"响应式"
const userName = ref('admin');

function updateName(name: string) {
  userName.value = name;  // 修改后，用到 userName 的视图自动更新！
}
```

### 1.3 组件 = 类，实例 = 对象

```
Vue 组件            ≈  Java 类
组件的 data/ref     ≈  类的字段
组件的 methods      ≈  类的方法
组件的 props        ≈  构造函数的参数
组件的 emits        ≈  回调接口/监听器
组件实例            ≈  类的实例对象
```

## 3. 核心技术栈精要

### 3.1 Vue3 Composition API（30 分钟上手）

#### 3.1.1 核心概念：setup 脚本

```vue
<script setup lang="ts">
// 这整个脚本就是在"setup"函数中
// 类似 Spring 的 @PostConstruct，组件创建时执行

// ========== 1. 响应式数据（类似类的字段，但自动触发更新）==========

// ref：基本类型的响应式包装
const count = ref(0);           // 像 AtomicInteger
const userName = ref('admin');  // 像 String 但会自动更新视图
const user = ref<User>({        // 对象也用 ref
  id: 1,
  name: '张三'
});

// reactive：仅用于对象（不推荐混用，项目主要用 ref）
const state = reactive({
  count: 0,
  userName: 'admin'
});

// computed：计算属性（类似 getter，有缓存）
const fullName = computed(() => {
  return `${user.value.firstName} ${user.value.lastName}`;
});

// ========== 2. 方法（类似类的方法）==========
function increment() {
  count.value++;  // ref 需要用 .value 访问
}

async function fetchUserData() {
  const { data } = await fetchGetUserInfo();
  user.value = data;
}

// ========== 3. 生命周期（类似 Servlet 生命周期）==========

// onMounted ≈ @PostConstruct
onMounted(() => {
  console.log('组件挂载完成');
  fetchUserData();  // 通常在此时加载数据
});

// onUnmounted ≈ @PreDestroy
onUnmounted(() => {
  console.log('组件即将销毁');
  // 清理工作：取消定时器、移除事件监听等
});

// watch ≈ 监听器/观察者模式
watch(
  () => user.value.id,  // 监听的数据源
  (newVal, oldVal) => { // 变化时的回调
    console.log(`用户ID变化: ${oldVal} -> ${newVal}`);
  }
);

// watchEffect ≈ 自动追踪依赖的 watch
watchEffect(() => {
  // 自动追踪用到的响应式数据
  console.log(`当前计数: ${count.value}`);
});
</script>
```

#### 3.1.2 与 Options API 对比

```vue
<script lang="ts">
// 旧写法（Options API）- 像 Vue2
export default {
  data() {
    return { count: 0 }
  },
  methods: {
    increment() { this.count++ }
  }
}
</script>

<script setup lang="ts">
// 新写法（Composition API）- 推荐
const count = ref(0);
function increment() { count.value++ }
</script>
```

**建议**：项目全部使用 `<script setup>` 语法，简洁直观。

### 3.3 Pinia 状态管理（类比 Spring 的 Service + Session）

#### 3.3.1 Store 定义

```typescript
// src/store/modules/auth.ts
import { defineStore } from 'pinia';

// 类似 @Service
export const useAuthStore = defineStore('auth', () => {
  // ========== State（类似类的字段）==========
  const token = ref('');
  const userInfo = reactive<Api.Auth.UserInfo>({
    userId: '',
    userName: '',
    roles: [],
    buttons: []
  });

  // ========== Getters（类似 getter 方法，有缓存）==========
  const isLogin = computed(() => !!token.value);
  const isAdmin = computed(() => userInfo.roles.includes('R_ADMIN'));

  // ========== Actions（类似 Service 方法）==========
  async function login(userName: string, password: string) {
    const { data } = await fetchLogin(userName, password);
    token.value = data.token;
    localStorage.setItem('token', data.token);  // 持久化
  }

  function logout() {
    token.value = '';
    localStorage.removeItem('token');
  }

  // ========== 暴露给外部使用 ==========
  return {
    token,
    userInfo,
    isLogin,
    isAdmin,
    login,
    logout
  };
});
```

#### 3.3.2 Store 使用

```vue
<script setup lang="ts">
import { useAuthStore } from '@/store/modules/auth';

// 获取 Store 实例（类似 @Autowired）
const authStore = useAuthStore();

// 访问 State（自动响应式）
console.log(authStore.token);
console.log(authStore.isLogin);  // Getter

// 调用 Action
async function handleLogin() {
  await authStore.login('admin', '123456');
  // 登录成功后，所有用到 authStore 的地方自动更新
}
</script>

<template>
  <!-- 直接在模板中使用 -->
  <div v-if="authStore.isLogin">
    欢迎, {{ authStore.userInfo.userName }}
  </div>
</template>
```

**关键理解**：

- Store 是**单例模式**，全局只有一个实例
- State 修改后，所有使用它的地方**自动更新**
- 可以跨组件共享状态（类似全局变量，但类型安全）

## 4. 实战开发流程

### 4.1 新增一个管理页面（完整流程）

以"商品管理"为例：

#### Step 1: 定义 API 接口

```typescript
// src/service/api/product.ts
export function fetchGetProductList(params?: Api.Product.ProductSearchParams) {
  return request<Api.Product.ProductList>({
    url: '/product/list',
    method: 'get',
    params
  });
}

export function fetchAddProduct(data: Api.Product.ProductCreateParams) {
  return request<boolean>({
    url: '/product/add',
    method: 'post',
    data
  });
}

export function fetchUpdateProduct(data: Api.Product.ProductUpdateParams) {
  return request<boolean>({
    url: '/product/update',
    method: 'put',
    data
  });
}

export function fetchDeleteProduct(id: number) {
  return request<boolean>({
    url: `/product/delete/${id}`,
    method: 'delete'
  });
}
```

#### Step 2: 定义类型

```typescript
// src/typings/api.d.ts（在已有 namespace 中添加）
declare namespace Api {
  namespace Product {
    interface Product {
      id: number;
      name: string;
      price: number;
      stock: number;
      status: '1' | '2';
      createTime: string;
    }

    type ProductSearchParams = Partial<
      Pick<Product, 'name' | 'status'> & Common.CommonSearchParams
    >;

    type ProductList = Common.PaginatingQueryRecord<Product>;

    type ProductCreateParams = Omit<Product, 'id' | 'createTime'>;

    type ProductUpdateParams = Partial<Product> & { id: number };
```

#### Step 3: 创建页面组件

```vue
<!-- src/views/manage/product/index.vue -->
<template>
  <div class="p-4">
    <!-- 搜索表单 -->
    <ACard class="mb-4">
      <AForm layout="inline" :model="searchForm">
        <AFormItem label="商品名称">
          <AInput v-model:value="searchForm.name" placeholder="请输入" />
        </AFormItem>
        <AFormItem label="状态">
          <ASelect v-model:value="searchForm.status" style="width: 120px">
            <ASelectOption value="">全部</ASelectOption>
            <ASelectOption value="1">上架</ASelectOption>
            <ASelectOption value="2">下架</ASelectOption>
          </ASelect>
        </AFormItem>
        <AFormItem>
          <AButton type="primary" @click="handleSearch">查询</AButton>
          <AButton class="ml-2" @click="handleReset">重置</AButton>
        </AFormItem>
      </AForm>
    </ACard>

    <!-- 操作按钮 -->
    <ACard class="mb-4">
      <AButton type="primary" @click="handleAdd">新增商品</AButton>
    </ACard>

    <!-- 数据表格 -->
    <ACard>
      <ATable
        :columns="columns"
        :data-source="dataList"
        :loading="loading"
        :pagination="pagination"
        @change="handleTableChange"
      >
        <template #bodyCell="{ column, record }">
          <template v-if="column.key === 'action'">
            <AButton type="link" @click="handleEdit(record)">编辑</AButton>
            <AButton type="link" danger @click="handleDelete(record)">删除</AButton>
          </template>
          <template v-if="column.key === 'status'">
            <ATag :color="record.status === '1' ? 'success' : 'default'">
              {{ record.status === '1' ? '上架' : '下架' }}
            </ATag>
          </template>
        </template>
      </ATable>
    </ACard>

    <!-- 新增/编辑弹窗 -->
    <ProductModal
      v-model:visible="modalVisible"
      :data="currentRecord"
      @success="handleSearch"
    />
  </div>
</template>

<script setup lang="ts">
import { ref, reactive, onMounted } from 'vue';
import { message, Modal } from 'ant-design-vue';
import { fetchGetProductList, fetchDeleteProduct } from '@/service/api/product';
import ProductModal from './modules/product-modal.vue';

// ========== 表格列定义 ==========
const columns = [
  { title: 'ID', dataIndex: 'id', key: 'id' },
  { title: '商品名称', dataIndex: 'name', key: 'name' },
  { title: '价格', dataIndex: 'price', key: 'price' },
  { title: '库存', dataIndex: 'stock', key: 'stock' },
  { title: '状态', dataIndex: 'status', key: 'status' },
  { title: '创建时间', dataIndex: 'createTime', key: 'createTime' },
  { title: '操作', key: 'action', fixed: 'right', width: 150 }
];

// ========== 响应式数据 ==========
const loading = ref(false);
const dataList = ref<Api.Product.Product[]>([]);
const searchForm = reactive({
  name: '',
  status: ''
});
const pagination = reactive({
  current: 1,
  pageSize: 10,
  total: 0,
  showSizeChanger: true,
  showTotal: (total: number) => `共 ${total} 条`
});
const modalVisible = ref(false);
const currentRecord = ref<Api.Product.Product | null>(null);

// ========== 方法 ==========
async function loadData() {
  loading.value = true;
  try {
    const { data } = await fetchGetProductList({
      current: pagination.current,
      size: pagination.pageSize,
      name: searchForm.name,
      status: searchForm.status
    });

    if (data) {
      dataList.value = data.records;
      pagination.total = data.total;
    }
  } finally {
    loading.value = false;
  }
}

function handleSearch() {
  pagination.current = 1;
  loadData();
}

function handleReset() {
  searchForm.name = '';
  searchForm.status = '';
  handleSearch();
}

function handleTableChange(pag: any) {
  pagination.current = pag.current;
  pagination.pageSize = pag.pageSize;
  loadData();
}

function handleAdd() {
  currentRecord.value = null;
  modalVisible.value = true;
}

function handleEdit(record: Api.Product.Product) {
  currentRecord.value = record;
  modalVisible.value = true;
}

function handleDelete(record: Api.Product.Product) {
  Modal.confirm({
    title: '确认删除',
    content: `确定删除商品 "${record.name}" 吗？`,
    async onOk() {
      const { error } = await fetchDeleteProduct(record.id);
      if (!error) {
        message.success('删除成功');
        loadData();
      }
    }
  });
}

// ========== 生命周期 ==========
onMounted(() => {
  loadData();
});
</script>
```

```vue
          style="width: 100%"
        />
      </AFormItem>
      <AFormItem label="状态" name="status">
        <ARadioGroup v-model:value="formData.status">
          <ARadio value="1">上架</ARadio>
          <ARadio value="2">下架</ARadio>
        </ARadioGroup>
      </AFormItem>
    </AForm>
  </AModal>
</template>

<script setup lang="ts">
import { ref, watch, computed } from 'vue';
import { message } from 'ant-design-vue';
import { fetchAddProduct, fetchUpdateProduct } from '@/service/api/product';

interface Props {
  visible: boolean;
  data: Api.Product.Product | null;
}

const props = defineProps<Props>();
const emit = defineEmits<{
  'update:visible': [value: boolean];
  success: [];
}>();

const formRef = ref();
const submitting = ref(false);
const formData = reactive({
  id: undefined as number | undefined,
  name: '',
  price: 0,
  stock: 0,
  status: '1' as '1' | '2'
});

const isEdit = computed(() => !!props.data);

const rules = {
  name: [{ required: true, message: '请输入商品名称' }],
  price: [{ required: true, message: '请输入价格' }],
  stock: [{ required: true, message: '请输入库存' }]
};

// 监听 data 变化，回填表单
watch(
  () => props.data,
  (newVal) => {
    if (newVal) {
      Object.assign(formData, newVal);
    } else {
      formRef.value?.resetFields();
    }
  },
  { immediate: true }
);

async function handleSubmit() {
  await formRef.value.validate();

  submitting.value = true;
  try {
    const api = isEdit.value ? fetchUpdateProduct : fetchAddProduct;
    const { error } = await api(formData);

    if (!error) {
      message.success(isEdit.value ? '编辑成功' : '新增成功');
      emit('update:visible', false);
      emit('success');
    }
  } finally {
    submitting.value = false;
  }
}
</script>
```

## 8. 踩坑指南

### 8.1 响应式数据丢失

```typescript
// ❌ 问题：直接解构会丢失响应式
const user = reactive({ name: '张三', age: 20 });
const { name } = user;  // name 不是响应式的

// ✅ 解决：使用 toRefs
import { toRefs } from 'vue';
const { name, age } = toRefs(user);

// 或保持原始引用
const userName = computed(() => user.name);
```

### 8.2 数组/对象更新

```typescript
const list = ref([1, 2, 3]);

// ✅ 正确更新数组
list.value.push(4);           // OK
list.value = [...list.value, 4];  // OK
list.value[0] = 10;           // OK（Vue3 支持）

const user = ref({ name: '张三' });

// ✅ 正确更新对象
user.value.name = '李四';     // OK
user.value = { ...user.value, name: '李四' };  // OK
```

### 8.3 异步获取数据后表单不显示

```vue
<script setup lang="ts">
// ❌ 问题：先创建了响应式对象，再异步获取数据
const formData = reactive({ name: '' });

onMounted(async () => {
  const { data } = await fetchUserInfo();
  formData = data;  // ❌ 错误！不能重新赋值 reactive 对象
});

// ✅ 解决1：使用 ref
const formData = ref({ name: '' });
onMounted(async () => {
  const { data } = await fetchUserInfo();
  formData.value = data;
});

// ✅ 解决2：使用 Object.assign
onMounted(async () => {
  const { data } = await fetchUserInfo();
  Object.assign(formData, data);
});
</script>
```

### 8.4 TypeScript 类型报错

```typescript
// ❌ 问题：可能为 undefined
const user = userList.value.find(u => u.id === 1);
console.log(user.name);  // 报错：user 可能为 undefined

// ✅ 解决1：类型断言
console.log((user as User).name);

// ✅ 解决2：可选链
console.log(user?.name);

// ✅ 解决3：类型守卫
if (user) {
  console.log(user.name);  // TypeScript 知道这里 user 不为 undefined
}
```
